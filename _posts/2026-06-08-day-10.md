what_i_learned: |
  In today’s learning, I was introduced to the concept of a 1D Convolutional Neural Network (1D-CNN) and how it is used for time-series data like ECG signals. A 1D-CNN is a deep learning model that processes data along one dimension, usually time, which makes it very useful for signals that change over time such as heartbeats. The main idea is that instead of manually selecting features from the data, the model automatically learns patterns directly from raw input. For ECG data, the input shape is usually (batch, 12, 1000), which represents 12 heart leads recorded over 1000 time steps. The model then applies layers like Conv1d, Batch Normalization, ReLU activation, and MaxPooling to gradually extract meaningful patterns. Each layer helps the model focus on different levels of features, from simple signal changes to more complex cardiac patterns. Finally, the extracted features are flattened and passed into a fully connected layer that produces predictions for different heart conditions. I also learned how the full 1D-CNN architecture works as a complete pipeline from input to prediction. The ECG signal first enters the model and goes through convolution blocks that slide small filters across the signal to detect local patterns such as spikes, dips, or irregular rhythms. Each convolution layer helps the model learn more advanced features as the data moves deeper through the network. Max-pooling is used after convolution to reduce the size of the data while keeping the most important information, which also helps reduce computation. After several layers, the output is flattened into a one-dimensional vector so it can be processed by the final fully connected layer. This final layer maps the learned features into class scores that represent different cardiac conditions. I also explored how PyTorch implements this process using a training loop that includes forward propagation, loss calculation, backpropagation, and parameter updates using an optimizer.

blockers: |
  One blocker I experienced during this lesson is understanding how tensor shapes change at each stage of the CNN. It can be confusing to track how the input shape (12, 1000) gets transformed after each convolution and pooling layer. Every layer changes the dimensions slightly, and it is easy to lose track of the correct output size. Another challenge is understanding how stride, padding, and kernel size all affect the final output dimensions. These concepts are important because even a small mistake in calculation can cause shape mismatch errors in the model. I also had to slow down when reading the PyTorch code because concepts like nn.Sequential, forward functions, and layer stacking are still new to me. Overall, the biggest difficulty is connecting the mathematical operations behind CNNs with how they are actually written in code.

what_i_learned: |
  During today’s session, I learned that the input() function in Python is used to read and return information directly from the terminal. This allows a program to interact with a user by storing whatever they type into a variable, which can then be used later in the code. For example, writing string_from_user = input() lets the computer capture user input and store it as a string. I also learned that printing that variable displays exactly what the user entered, making it a simple way to build interactive programs. This concept is important because it is the foundation of how many real-world applications collect data from users. Overall, input() helps bridge communication between humans and computers in a very direct way. I also learned about tokenization and embeddings, which are key steps in how language models understand text. Tokenization is the process of breaking down sentences into smaller pieces called tokens, which can be words, subwords, or punctuation. For example, “ChatGPT is smart” can be split into tokens like “Chat,” “GPT,” “is,” and “smart.” After tokenization, the model converts these pieces into numbers called embeddings because computers only understand numerical data. These embeddings represent the meaning of words in a way the model can process and learn from. I also learned that large language models predict the next token in a sequence, similar to advanced autocomplete, using context from previous words. This is how models can generate sentences like “The sky is blue” after seeing “The sky is.” Additionally, neural networks are inspired by the human brain and help models learn patterns from data. Finally, prompt engineering is important because the way a question is asked affects the quality of the model’s response, meaning clearer prompts lead to better answers.

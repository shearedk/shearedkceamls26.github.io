what_i_learned: |
  Today was my first day learning about multi-label classification, and it introduced me to a completely different way of thinking about machine learning predictions. Before today, I mainly understood classification as choosing one category from several possible options, but I learned that some real-world problems require multiple labels at the same time. Using ECG data as an example helped me understand this concept better because a patient can have more than one heart condition simultaneously. I learned the difference between multi-class and multi-label classification and why they require different output layers and loss functions. One of the most important concepts I learned was that multi-class problems use Softmax and CrossEntropyLoss, while multi-label problems use Sigmoid and BCEWithLogitsLoss. Seeing actual patient examples with different combinations of conditions made the lesson much easier to understand and showed me why multi-label classification is necessary in healthcare applications. Another thing I learned was how to modify a neural network model so that it can predict multiple labels instead of just one. I learned that the output layer must be changed to produce five outputs, one for each ECG superclass. We also discussed the importance of using weighted loss functions to help the model pay more attention to rare conditions that do not appear as often in the dataset. In addition, I learned about several evaluation metrics that are specifically useful for multi-label classification. Metrics such as Macro F1, Micro F1, Hamming Loss, and Subset Accuracy all measure performance differently and provide unique insights about model quality. Learning these evaluation methods helped me realize that accuracy alone is not always enough when assessing machine learning models, especially when dealing with multiple labels and imbalanced data.

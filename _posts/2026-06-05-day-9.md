what_i_learned: |
  Computer vision is a branch of Artificial Intelligence (AI) that allows computers to “see” and understand images and videos using cameras and data. I learned that an algorithm is a set of step-by-step instructions that helps a computer complete a task, while machine learning (ML) helps systems improve by learning from data over time. A model is what is created when an algorithm is trained on data so it can make predictions or decisions. In image classification, the computer is trained using labeled examples so it can learn what different objects look like. For example, if it sees many images of a person like LeBron James labeled correctly, it learns to recognize him in new images. Computer vision systems also break images into pixels and look for patterns like shapes, edges, and colors to understand what is inside the image. I also learned how object detection works, which is not just identifying what is in an image but also where it is located. The system uses bounding boxes to draw rectangles around detected objects, such as people or objects in a scene. YOLO (You Only Look Once) is a fast detection model that scans an image one time instead of multiple passes, making it efficient for real-time use. YOLO also divides images into grids to help locate objects and then predicts bounding boxes and labels. Tools like OpenCV help computers load and process images in different formats, such as color or unchanged images with transparency. Overall, I now understand how AI systems combine cameras, algorithms, and models to interpret visual information.

blockers: |
  One challenge I faced was fully understanding how all the parts of computer vision connect together, especially how pixels turn into meaningful predictions. At first, the idea of breaking images into thousands of tiny pixel values felt overwhelming and abstract. I also had difficulty understanding how YOLO can process an image so quickly while still detecting multiple objects accurately. Some of the technical terms like bounding boxes, grids, and model training required me to review the material multiple times. I also felt unsure about how OpenCV functions like cv2.imread() fit into the bigger computer vision pipeline. Even though I understood each concept individually, putting them together into a full workflow was confusing. 

reflection: |
